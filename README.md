## 레포트 내용

1. 기존 은행의 고객 자산 관리 방법
    
    기존 은행에는 돈을 계정과 계좌로 나누어 관리하고 있는것으로 알고 있습니다. 계정이 하나 있으면, 그 계정의 수많은 고객의 계좌가 연결되어 관리되고 있습니다.
    
    이런 시스템 아래에서 은행 전산에 어떠한 거래가 발생 시 적어도 두 계정 사이에서의 분개가 발생합니다. (대차대조표상에 차변과 대변의 합계가 같도록 원장(계정) 수정)
    
    또한 각 계정은 마스터 계정 (e.g. 은행 전체 원화예금계정은 각 지점별 원화예금계정의 합) 에 종속되어 하위 계정이 수정될시 마스터 계정 또한 즉시 적용되도록 되어 있습니다. 
    
2. 블록체인 지갑(wallet)은 무엇인가? 
    
    대표적인 블록체인 시스템인 비트코인, 이더리움과 같은경우 ECDSA(타원곡선 암호)와 같은 비대칭 암호화 방식을 사용해서 공개키는 주소(address,다만 주소자체가 공개키는 아니라 가공의 과정을 거친다.)로, 비밀키는 지갑의 소유권 인증의 용도(password)로 사용한다. 
    
    비밀번호는 무엇을 의미하냐.. 공부한 내용을 바탕으로 이해한 바는 이렇다. 내 공개키(주소)로 거래한 거래내역에 대해서, ‘내가’ 했음을 ‘디지털서명’할게.! → 이 거래는 내가 한게 맞아.! 유효한거래야! 라는 뜻이다.  비밀키로 암호화했으니, 다른 Node(검증자)들이 내가 요청한 거래가 유효한지 나의 공개키를 통해 알수 있다.
    
    아래는 내가 정리한 내용에 대한 AI의 보강이다.
   ```
    "비밀키로 암호화했으니 공개키로 알 수 있다"는 표현은 절반만 맞습니다. 현대의 디지털 서명은 전체 데이터를 암호화하지 않습니다.
    
    1. **해싱:** 거래 내역(A가 B에게 1BTC 송금)을 해시 함수에 넣어 짧은 '지문'을 만듭니다.
    2. **서명:** 이 지문을 **비밀키**로 연산하여 **'디지털 서명'**값을 생성합니다.
    3. **검증:** 검증자(Node)는 **[거래 내역 + 서명 + 나의 공개키]**를 받아서, "이 서명이 이 공개키의 주인에 의해 만들어진 게 맞는가?"를 수학적으로 확인합니다.
    
    > **중요:** 서명을 공개키로 '복호화'해서 데이터를 꺼내는 게 아니라, 서명 자체가 유효한지를 '확산(Verification)'하는 과정입니다.
    ```
    
3. 수탁형 지갑과 비수탁형 지갑의 차이
    
    수탁형 지갑은 제 3자(서비스제공자)가 비밀키를 갖고 있으며, 나는 아이디와 비밀번호를 통해 내 지갑에 접근할 수 있다. (업비트, 커스터디 등등), → 사용자는 자산에 대한 ‘청구권’만을 갖고 있으며, 제어권은 피수탁자에게 존재한다.
    
    비수탁형 지갑은 MetaMask등의 지갑 App, 혹은 비트코인 풀노드와 같은 앱으로서, 내가 직접 비밀키를 관리하며, 비밀키를 유실하게 되면 지갑에 대한 소유권을 바로 잃게 된다. 
    
4. 은행의 계정/인증체계와 블록체인 지갑의 차이점
    
    은행은 나의 ID, 비밀번호, 공인인증서 등등의 인증수단을 통해 자체 서버(혹은 외부서버)에서 계좌의 소유자임을 인증하며, 이 방식은 서버 저장값과, 내가 입력한 방식이 같은지 검증하는 방식으로 진행된다. 
    
    블록체인의 ‘비수탁형’ 지갑의 경우는, 비밀키 자체가 계정의 소유권 자체를 의미하며, 검증또한 수학적으로 계산하여 진행된다. 
    
    다만, 수탁형지갑의 경우, 지갑의 소유권에 대한 관리(청구권)을 기존의 전통적인 계정/인증체계와 같이 관리하게 된다.
    
5. 금융권은 이러한 구조를 어디까지 수용할 수 있을까?, 그리고 어떠한 방향으로 나아가야할까>?
    
    비트코인이 주장하는 익명성, 탈중앙화 등등의 가치는 우리 은행에서 제공 (실명성, 중앙화) 하는 가치와 상반되나.. 블록체인이라는 개념이 등장한 이상, 세상 모든 금융서비스를 제공하려 하는 기존 금융권 기업들은 어디까지 블록체인을 수용해야할지 고민이 필요할 것이다..
    
    블록체인이 제공하는 정합성 (데이터 위변조 가능성 x, 특히 요새 주목하는 내부통제에 있어 탁월함), 실시간 업무 자동화 (스마트 컨트랙트를 올리면 중개자 없이 금융 서비스를 자동으로 실행할 수 있다.) 정도는 기존 금융권이 기술적으로 받아들일 수 있을것이다. 
    
    또한 은행은 기보유중인 튼튼한 본인인증/관리 프로세스가 존재한다. 이러한 프로세스를 코인을 보유하고 싶어하나, 비밀키 관리에 두려움을 느끼는 고객층에게 제공할수도 있지않을까. 일종의 블록체인/코인/토큰 예금서비스가 제공되는 것..
